Quality Improvement Measures and Billing

July 2024 to June 2025

1 Introduction

Comparing Quality Improvement Measure (QIM) outcomes between practices which predominantly bulk-bill (80% or more) and ‘other’ practices during the time period July 2024 to June 2025.

The Quality Improvement Measures are intended to support a regional and national understanding of chronic disease management in areas of high need1. Though the QIMs are not designed to assess individual general practices or general practitioner performance, general practices are encouraged to improve their Quality Improvement Measures relative to their peers.

General practices are incentivised, through Practice Incentive Program payments, to report Quality Improvement Measures to Primary Health Networks. As of 2024–2025 General practices have not been incentivised to meet any specified performance level in any of the Quality Improvement Measures.

Data derived from Cubiko Touchstone, February 20262.

  "./Quality Improvement Measures vs Billing.xlsx",

# 2) Reshape: Centile values -> separate columns
df_wide <- df %>%
  # keep only the five centiles we need (defensive)
  filter(Centile %in% c(10, 25, 50, 75, 90)) %>%
  # ensure a sensible order
  mutate(Centile = factor(Centile, levels = c(10, 25, 50, 75, 90))) %>%
  # pivot to wide: one row per grouping combo, columns for each centile’s Proportion
  pivot_wider(
    id_cols = c(Billing, MMM, PHN, QIM, `QIM description`),
    names_from  = Centile,
    values_from = Proportion,
    names_glue  = "p{Centile}",               # -> p10, p25, p50, p75, p90
    values_fn   = list(Proportion = ~ mean(.x, na.rm = TRUE)), 
    # ^ if duplicates exist within a group+centile, take the mean; 
    #   replace with first(.x) or sum(.x) if preferred
    values_fill = NA_real_
  )

# Result: df_wide has columns:
# Billing, MMM, PHN, QIM, QIM description, p10, p25, p50, p75, p90

# Compare all QIMs for two PHNs
#   NWMPHN, EMPHN
# 1) Filter to PHN, Billing, build QIM labels
df_box <- df_wide %>%
  filter(
    PHN %in% c("NWMPHN", "EMPHN"),
    Billing %in% c("BB 80+", "Other")
  ) %>%
  mutate(
    QIM_label = paste0(QIM, ". ", `QIM description`),
    QIM_label = fct_reorder(QIM_label, QIM),
    Billing   = factor(Billing, levels = c("BB 80+", "Other")),
    PHN       = factor(PHN, levels = c("NWMPHN", "EMPHN"))
  ) %>%
  # 2) Ensure exactly one row per (QIM × Billing × PHN) for identity boxplots
  group_by(QIM_label, Billing, PHN) %>%
  summarise(
    p10 = mean(as.numeric(p10), na.rm = TRUE),
    p25 = mean(as.numeric(p25), na.rm = TRUE),
    p50 = mean(as.numeric(p50), na.rm = TRUE),
    p75 = mean(as.numeric(p75), na.rm = TRUE),
    p90 = mean(as.numeric(p90), na.rm = TRUE),
    .groups = "drop"
  ) %>%
  # 3) Build a unique x for each box to avoid dodging ambiguity
  mutate(x_ib = interaction(Billing, PHN, lex.order = TRUE))

# verify uniqueness: should be 1 everywhere
